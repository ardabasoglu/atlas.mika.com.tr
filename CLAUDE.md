# Atlas — AI and design context

## Design Context

### Users
Mika staff use Atlas to manage leads, deals, projects, and campaigns. The product is an internal information and CRM platform. The job to be done: run pipelines and projects with confidence, without visual noise getting in the way.

### Brand Personality
**Professional, clear, reliable.** Voice and tone should feel trustworthy and competent. The interface should inspire confidence and minimize distraction—no playfulness or clutter.

### Aesthetic Direction
- **Reference:** Stripe—clean layout, clear hierarchy, restrained use of color, plenty of whitespace, legible typography.
- **Avoid:** Playful tone, cluttered screens, decorative excess. Keep density and ornament minimal.
- **Theme:** Support both light and dark (as in current implementation); default can follow system or stay light for a “document-like” feel.

### Design Principles
1. **Clarity over cleverness** — Every element should have a clear purpose. Prefer obvious over clever.
2. **Confidence through restraint** — Use the existing palette and components consistently. Avoid extra colors or styles that don’t add meaning.
3. **Minimal distraction** — Reduce visual noise: generous spacing, clear headings, and a calm, professional tone (aligned with Stripe-like reference).
4. **Trustworthy and reliable** — Copy and states (errors, empty, loading) should be direct and helpful. No jokes or casual language that could undermine trust.
5. **Professional consistency** — Stick to Outfit + Geist Mono, semantic tokens (primary, muted, destructive), and the current radius scale. New UI should feel like part of the same system.
